## 已完成：更新进度

Windows 自动更新的非阻塞下载进度窗口已经实现。窗口独立于便签，可拖动、最小化并从任务栏恢复；下载期间便签和托盘继续工作。更新控制器使用显式阶段和 operation ID 防止迟到事件、重复下载与重复安装提示，关闭全部便签后 Windows 仍由托盘常驻。

真机故障的根因不是 updater 没有发送进度，而是 `sandbox: true` 的进度 preload 曾被构建为 ESM，导致 bridge 没有执行。当前构建强制两个 preload 输出 CommonJS，主窗口加载 `preload.cjs`，进度窗口加载 `updateProgressPreload.cjs`。

## 已完成并合入：便签命名

### 产品目标

每张便签可以有一个名字，帮助用户区分项目、工作和生活等不同用途；名字只是安静的识别锚点，不引入分类、标签、文件夹、筛选或工作区，也不新增一行标题表单。

### 已确定交互

- 名字写在顶部拖拽条中间，复用当前 `status-label` 的位置。
- 空名字默认不显示，保持现有界面干净。
- 顶部中央保留一个固定、透明的命名热区；只有鼠标进入该热区时，才显示名称或“`双击命名`”。正文、清单、工具栏、抓手和其余顶栏不会触发提示。
- 双击中间区域后，原地变成无边框输入框，自动聚焦并全选。
- `Enter` 或失焦保存，`Esc` 取消并恢复编辑前的名字。
- 保存成功后名称短暂显示约 1.5 秒，随后恢复隐藏；完整名字通过 tooltip 查看。
- 名称热区和编辑框始终使用 `-webkit-app-region: no-drag`，避免 macOS 把普通双击吞成标题栏缩放。左侧 40px 抓手和其余顶栏仍可拖动窗口。
- 有名字时系统窗口标题使用该名称，空名字回退为“悬浮便签”；renderer 页面标题不能覆盖主进程设置的窗口标题。
- 临时动作状态约 2 秒后自动清除；终态“读取失败”保持显示。编辑器优先于无关临时状态，不能被后台状态卸载。
- 空白或只含空格的输入保存为空名字；输入在可信边界统一 `trim`，并限制为最多 60 个 Unicode code point。
- 不增加常驻重命名图标。
- 本版本不做便签右键菜单。以后完整实现时再把“重命名 / 新建 / 颜色 / 透明度 / 置顶 / 删除”一起放入，避免现在出现只有一项的不完整菜单。

显示优先级为：

```text
正在编辑 > 临时状态 > 命名热区 hover / 保存确认 > 空白
```

### 实现与验证状态

- 复用既有 `name` 数据字段和 `sticky-notes:update-name` IPC，不需要数据迁移。
- 主进程在保存边界统一归一化名称，并在建窗和改名后同步系统窗口标题。
- renderer 使用独立命名状态逻辑处理开始、提交、取消、IME 和保存失败，避免 Enter 与 blur 重复提交。
- 固定透明命名热区消除了动态切换 `drag / no-drag` 引发的 Electron 原生命中竞态。
- 自动化闸门：39 个测试文件、206 个测试通过；`npm run build`、`npm audit --omit=dev`、tracked 与 untracked diff whitespace 检查均通过。
- 最新完整 diff 已完成 Codex 主 agent 本地 review 和 Opus 独立只读 review；最终无 HIGH 或 MEDIUM finding，最后一轮 Opus 对新增改动为 `NO_FINDINGS`。
- 2026-07-12 用户已在当前 Mac 开发版完成物理验收并明确回复“通过”：正文、清单、工具栏和抓手不会误触发提示；顶部中央热区悬停显现；普通单指双击进入命名且不触发系统标题栏缩放；Enter、失焦、Esc、保存后约 1.5 秒短显、拖窗和重启持久化均符合预期。

非阻塞残留风险：接近 200px 的窄窗主要依赖左侧抓手拖动，空态提示可能被裁切；60 code point 截断可能在 ZWJ、旗帜或肤色等多码点字形簇中间截断；名称保存失败后会恢复旧名称并丢弃本次 draft。以上均已在 review 中判定为 LOW，不得描述成已消除。

`main` 是下一版本的集成线，不等于已发布版本。当前发布顺序为：

```text
已发布 v0.1.10
    |
main + 更新进度 + 便签命名 + 工具链安全修复
    |
统一升版 0.1.11、打包、真机验证
```

1. `package.json`、lockfile 和 `[0.1.11]` 变更日志保持一致。
