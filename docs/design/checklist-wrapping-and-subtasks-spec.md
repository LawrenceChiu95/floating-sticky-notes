# 待办完整展示与子任务 Product Spec

> 日期：2026-07-15
> 状态：功能一与功能二均已随 0.1.14 发布
> 范围：悬浮便签中的待办清单

## 一句话决策

先让每条待办都能完整、自然地显示；再增加一层可选子任务，帮助用户把复杂目标拆成可执行步骤。两项能力分别开发、分别验收、可以分别发布。

## 用户问题

当前待办使用单行显示。便签变窄或文字较长时，后半段内容无法直接看到，用户也没有自然的方式查看全文。

这破坏了便签最基本的承诺：用户写下的信息应该随时可读，而不是因为窗口较窄而变得不可达。

与此同时，有些长内容并不只是“一句话比较长”，而是一个目标包含多个可以独立完成的步骤。仅靠换行可以保证信息完整，但不能帮助用户表达这些步骤之间的关系。

## 产品原则

1. **信息完整优先**：用户不需要为了看全内容而改写、缩短或拆分任务。这里的“完整”指不截断、不省略，并不把一条待办扩展成支持多个段落的富文本。
2. **默认保持简单**：不使用子任务时，体验仍然是现在的扁平待办清单。
3. **结构按需出现**：只有用户主动缩进或添加子任务后，才出现父子层级。
4. **系统不替用户宣布完成**：父任务代表最终目标，是否完成由用户判断；子任务只代表执行步骤。
5. **键盘优先，但能力不能隐藏**：Tab 提供高效操作，同时保留可发现的鼠标入口。
6. **避免变成项目管理器**：第一版只支持一层子任务，不加入备注、拖拽排序、多层嵌套、日期或负责人。

## 功能一：待办文字完整展示

### 目标

无论便签多窄、文字多长，用户都能直接阅读完整任务内容。

### 用户体验

- 任务文字根据便签宽度自然换行。
- 任务行随内容增加高度，不限制行数，不显示省略号。
- 多行任务的复选框和删除按钮与第一行文字对齐。
- 中文、英文、网址、订单号和连续数字都不会横向溢出。
- 便签内容较多时，整张便签继续上下滚动，不出现左右滚动。
- 文字换行只是视觉排版，不改变当前快速录入习惯：按 Enter 仍然创建下一条任务。
- 中文输入法按 Enter 确认候选词时，不会误创建下一条任务。
- 调整便签宽度后，任务内容立即重新排版。

### 不做什么

- 不限制为两行或三行。
- 不增加“展开全文”。
- 不要求用户把长文字改写成备注或子任务。
- 不允许在同一条任务里用 Enter 手动插入段落；每条待办仍然表达一个目标或动作。

### 验收标准

> 用户写入的任何任务文字都可以直接完整阅读；便签无横向滚动，既有新增、勾选、删除和键盘录入习惯不被破坏。

## 功能二：一层子任务

### 目标

当一个目标包含多个可独立执行的步骤时，用户可以快速把这些步骤组织到父任务下面，同时保持便签轻量。

### 信息模型

```text
□ 到家怡贷补丁需求                 ← 父任务：最终目标
  □ 完成剩余 10%                    ← 子任务：执行步骤
  □ 等待 APPLYD 确认
  □ 提交补丁并回归验证
```

- 父任务是用户最终要完成的目标。
- 子任务是帮助完成目标的可选步骤。
- 不做备注；进度说明若需要保留，可以直接写在任务文字中并自然换行。
- 第一版只允许父任务和子任务两层，不允许子任务继续拥有子任务。

### 完成逻辑

父任务和子任务的完成状态彼此独立：

- 子任务全部完成，不自动完成父任务。
- 完成父任务，不自动完成或取消子任务。
- 父任务完成后，只给父任务本身加完成样式；未完成子任务仍保持可见、可编辑、可勾选。
- “父任务已完成但仍有未完成子任务”是允许存在的状态，因为用户可能基于外部结果判断目标已经结束。
- 第一版不增加自动提醒、进度数字或“全部子任务已完成”的提示。
- 第一版不因父任务完成而自动折叠子任务，避免信息突然消失。

### 父子层级视觉

- 只有实际拥有至少一条子任务的父任务使用 `600` 字重；没有子任务的普通顶层事项保持现有常规字重。
- 加粗表达“这条事项统领下方步骤”，与子任务缩进和左侧引导线共同构成层级线索；不把所有顶层事项统一加粗，避免扁平清单整体变重。
- 父任务完成后仍保留结构字重，并叠加现有灰字与删除线；结构角色和完成状态彼此独立。
- 创建空子任务时父任务可以立即加粗；空子任务被放弃并自动移除后恢复常规字重，不为避免短暂变化引入额外状态。

### 创建子任务：键盘主操作

```text
Enter       创建下一条任务
Tab         把当前任务缩进为子任务
Shift+Tab   把当前子任务恢复为普通任务
```

具体规则：

- 用户可以先连续输入多条任务，再通过 Tab 建立父子关系。
- Tab 会把当前任务归到它紧邻上方的任务结构中：如果上一条是普通任务，就成为它的子任务；如果上一条已经是子任务，就加入该子任务所属的同一个父任务。
- 当前任务上方没有任何任务时，Tab 不产生变化。
- 子任务再次按 Tab 不继续缩进。
- 已经拥有子任务的父任务不能再被缩进，按 Tab 不产生变化，避免出现第三层。
- 子任务按 Shift+Tab 后恢复为普通任务，并保留原文字和完成状态。

### 创建子任务：可发现的鼠标入口

当鼠标悬停在父任务上，或该任务正被编辑时，在任务行右侧显示一个轻量 Plus 入口：

```text
□ 到家怡贷补丁需求                                      ＋
```

- 平时隐藏且不持续占用便签空间；隐藏时不得撑高任务间距或改变扁平清单的视觉密度。
- 鼠标悬停、任务进入编辑状态或入口通过键盘获得焦点时显示；不能做成只有鼠标悬停才能使用的隐藏能力。
- 从任务文字移动到入口的过程中，入口必须保持可见且可点击，不得因任务行丢失 hover 而消失。
- 点击后在父任务下方创建一个空子任务，并立即进入输入状态。
- 该入口与 Tab 产生完全相同的子任务结构。
- 子任务本身不显示“＋ 添加子任务”，因为第一版只支持一层。
- 保留“待办事项”作为空清单创建入口和空事项占位文案，不用 Tab / Shift+Tab 替换基础提示。
- “添加子任务”入口的 tooltip 同时提示 `Tab` 快捷键；文本框的无障碍名称补充 `Tab` / `Shift+Tab` 说明，但不在任务行内显示长段快捷键文案。
- 快捷键教学不应抢占任务正文空间或把编辑行变成工具栏；空清单仍只显示“待办事项”，已有子任务也不增加常驻快捷键横幅。

### 连续录入

- 在父任务上按 Enter：在该父任务的全部子任务之后创建下一条普通任务，不打断现有父子任务块。
- 在子任务上按 Enter：在当前子任务后创建同一父任务下的下一个子任务，方便连续拆解步骤。
- 在空子任务上再次按 Enter：移除空子任务，并把焦点移回父任务。
- 新建出的空子任务如果不填写内容便退出，自动移除，不留下空行。
- 普通任务按 Shift+Tab 不产生变化。
- 中文输入法组字期间，Enter 只确认输入，不创建新任务。

### 删除与取消层级

遵循“尽量不让用户意外丢失内容”的原则：

- 删除普通任务且它没有子任务：维持当前的直接删除行为。
- 删除父任务且它有子任务：只删除父任务，原有子任务自动提升为普通任务，并保持原顺序、文字和完成状态。
- 清空父任务并触发现有的空任务删除逻辑时，处理方式相同：子任务提升为普通任务。
- 删除单个子任务：只删除该子任务，不影响父任务和其他子任务。
- 空子任务按 Backspace：删除空子任务，并把输入焦点移到上一条可见任务。
- 非空子任务不会因为 Backspace 自动取消层级；取消层级统一使用 Shift+Tab，避免误操作。

### 排列规则

- 子任务始终紧跟在父任务下方。
- 多个子任务保持创建顺序。
- 第一版不支持拖拽排序、跨父任务移动或批量调整层级。
- 旧版已经存在的任务全部作为普通任务显示，不需要用户整理或迁移。
- 如果本地数据中的父子关系异常，优先保留文字：无法找到父任务的子任务会在数据修复时清除失效的层级关系，并作为普通任务保存，保证当前显示与下次重启一致。
- 保存和读取清单时，新增的父子关系必须被完整保留；单条异常任务不得导致整张清单保存失败或其他任务丢失。
- 父子层级必须能被辅助技术识别；“＋ 添加子任务”必须可以通过键盘获得焦点和操作。

### 验收标准

> 不使用子任务的用户不会感到清单变复杂；需要拆解目标的用户可以用 Enter、Tab、Shift+Tab 连续完成录入和调整，并且退出、重启和升级应用后，层级、文字、顺序与完成状态都保持不变。

## 为什么拆成两个开发分支

### 分支一

```text
fix/checklist-text-wrapping
```

只解决“内容看不完整”的基础问题。它可以独立验收、独立发布，不需要等待子任务。

### 分支二

```text
feat/checklist-subtasks
```

在功能一合入后，从最新主分支创建。它复用完整换行能力，并增加父子结构、快捷键和数据保存。

### 顺序

```text
main
  └─ fix/checklist-text-wrapping
       └─ 合入 main
            └─ feat/checklist-subtasks
```

这样可以避免：

- 为了等待子任务而延迟基础可用性修复。
- 两项改动互相掩盖问题。
- 子任务需要调整时，无法单独保留或发布换行修复。

## 发布与验证

### 功能一重点验证

- 默认宽度和最窄宽度下的长中文、长英文、网址和连续数字。
- 调整窗口宽度时重新换行。
- 多条长任务导致整张便签上下滚动。
- 中文输入法选词、Enter 新建、空任务退出、Backspace 和焦点移动；输入法确认候选词时不得误创建任务。
- Windows 与 macOS 实际应用中的显示和输入体验。

### 功能二重点验证

- Tab、Shift+Tab、Enter 和“＋ 添加子任务”的完整路径。
- 父子任务各自勾选，不互相修改。
- 删除父任务后，子任务安全提升。
- 旧版便签数据无损打开。
- 退出应用、重新启动和升级后，父子关系与顺序保持不变。
- 最窄便签下，父任务和子任务仍然完整换行并可滚动访问。

## 成功标准

1. 再没有“任务后半段看不到”的反馈。
2. 用户无需学习子任务，也能继续把便签当作简单清单使用。
3. 熟悉键盘的用户可以用 Enter + Tab 快速拆解任务，不需要离开输入流。
4. 子任务能力不会引入自动完成、隐藏内容或意外删除等替用户做决定的行为。
5. 两项能力均可独立发布、验证和回滚。
