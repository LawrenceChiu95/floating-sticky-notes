# 便签名称常驻与视觉层级设计

## 背景

`0.1.11` 支持为便签命名，并把名称同步到 Windows 任务栏预览和 Alt-Tab。当前便签正面的名称只在中央命名热区悬停、编辑中或保存成功后的短暂确认期显示；鼠标移开后重新隐藏。

真实用户把名称理解为便签的持续身份，认为“鼠标移开就消失”代表功能失效，同时反馈当前名称过于不显眼。原设计希望保持纸面安静，但把“视觉弱化”做成了“完全隐藏”，削弱了命名功能用于扫视区分便签的价值。

Windows 任务栏预览已经能正常跟随便签名称，不属于本次问题。

## 目标

- 已命名便签无需鼠标操作即可看到名称。
- 名称保持安静，不压过正文和清单内容。
- 常驻名称仍能提示用户可以双击编辑。
- 未命名便签不常驻显示操作提示。
- 名称保存失败时保留用户输入，允许直接重试。
- 保持现有拖动区域、工具按钮和窄窗口可用。

## 非目标

- 不增加独立标题行、重命名按钮或右键菜单。
- 不改变名称存储格式、60 个 Unicode code point 上限或系统窗口标题规则。
- 不改变 Windows 任务栏预览、Alt-Tab、应用身份或窗口分组方式。
- 不重构整个顶部工具栏，也不扩展到分类、标签、文件夹或工作区。
- 不处理整个便签在最低透明度下叠加任意桌面背景时的全局对比度问题。

## 已选视觉方案

### 已命名静止态

- 非空名称始终可见，不再依赖 hover 或保存确认 class 才恢复 opacity。
- 名称使用 `12px`、`font-weight: 600`。
- 名称使用统一的克制墨色 `rgba(43, 42, 39, 0.82)`；现有五种可选便签色均为浅色，不引入动态前景色算法。
- 名称保持单行、省略号截断和完整内容 tooltip。
- 只调整名称本身，不放大或加粗共用的临时状态文案。

### 已命名悬停态

- 静止时只显示文字。
- 鼠标进入名称热区后增加 `rgba(43, 42, 39, 0.07)` 的圆角底色，提示该区域可以编辑。
- 保留文本鼠标指针和双击编辑，不增加图标、下划线或常驻按钮。

### 编辑态

- 输入框继续原地替换名称，不新增布局行。
- 输入框与常驻名称使用相同的 `12px / 600`，避免进入编辑时发生视觉跳变。
- 输入框维持无边框、透明外观；聚焦或编辑时使用与悬停态一致的弱底色。
- `Enter` 或失焦提交，`Esc` 取消并恢复编辑前的已保存名称。

### 未命名态

- 空名称静止时保持空白。
- 只有鼠标进入中央命名热区时，才显示常规字重、弱化颜色的“`双击命名`”，并使用同一弱底色。
- 清空名称保存成功后直接回到未命名空态，不额外显示“名称已移除”。

## 状态与数据流

显示优先级保持为：

```text
正在编辑或保存名称 > 临时状态 > 已保存名称 > 未命名 hover 提示 > 空白
```

1. 双击名称或空名称热区后，renderer 以当前已保存名称创建 draft，并聚焦、全选输入框。
2. 用户按 `Enter` 或让输入框失焦后，renderer 提交 draft。保存请求完成前保留输入框和 draft，并阻止重复提交。
3. 主进程继续在可信边界执行 trim 和 60 code point 截断，先持久化，再更新原生窗口标题，最后返回规范化后的名称。
4. 保存成功后，renderer 使用主进程返回值替换本地名称并退出编辑；非空名称立即进入常驻静止态，空名称进入未命名空态。
5. 保存失败后，renderer 保持编辑态和原 draft，解除提交锁，恢复输入焦点，并用短暂错误描边与 `aria-live` 文案“保存失败”反馈；用户可以修改后重试或按 `Esc` 取消。
6. 无关的临时状态不能卸载正在编辑或等待保存的名称输入框。未编辑时，临时状态可以短暂覆盖常驻名称，结束后恢复名称。

保存成功后不再需要“名称显示约 1.5 秒后重新隐藏”的确认状态或定时器。

## 顶栏与窄窗口约束

- 名称热区继续使用 `-webkit-app-region: no-drag`，最大宽度保持 `96px`，不因常驻而扩大不可拖动区域。
- 左侧 `40px` 抓手继续作为可靠拖动入口；其余可拖动顶栏行为保持不变。
- 名称区域允许随可用空间收缩并以省略号截断，不得覆盖抓手或右侧工具按钮。
- 完整名称继续通过 tooltip 查看。
- 必须人工检查 `200px` 最小宽度、`280px` 默认宽度和较宽窗口；极窄窗口可以更强地截断名称，但抓手和工具操作不能失效。

## 错误与无障碍

- 名称保存失败不能丢弃 draft，也不能静默回退后要求用户重新输入。
- 错误描边只作用于名称输入框，不改变整个顶栏或便签颜色；使用 `1px solid rgba(185, 28, 28, 0.55)`，用户再次修改或重试时清除。
- 错误描边提供可见反馈，同时通过现有 `aria-live` 区域播报“保存失败”。
- 保存等待期间输入框设置 `aria-busy="true"`，并阻止 Enter、blur 或双击触发第二次保存。
- 长名称继续保持 tooltip；键盘行为和输入法组合输入处理保持现状。

## 测试与验收

### 自动化

- 命名状态测试覆盖：非空名称常驻展示、空名称提示、保存成功使用规范化返回值、保存失败保留 draft 和编辑态、重复提交被阻止、`Esc` 恢复旧名称。
- renderer wiring 测试覆盖：已命名与未命名使用不同显隐 class，移除 1.5 秒隐藏确认逻辑，编辑器优先于无关临时状态。
- 样式测试覆盖：名称为 `12px / 600`、已命名默认可见、空名称默认隐藏、悬停弱底色、输入框与名称排版一致。
- 现有主进程名称归一化、持久化和原生窗口标题测试继续通过。

### 人工

- 在默认宽度确认名称常驻、视觉弱于正文但可扫视识别。
- 检查已命名 hover、双击、Enter、失焦、Esc、长名称省略和 tooltip。
- 检查未命名静止为空白、中央热区 hover 显示“双击命名”。
- 模拟保存失败，确认 draft 未丢失、错误反馈可见并可重试。
- 检查 `200px`、`280px` 和较宽窗口中的工具栏、抓手、名称截断和拖动行为。
- 检查五种便签颜色以及 `0.3`、默认值和 `1` 三档透明度。
- 在 Windows 与 macOS 开发版验证常驻、悬停、编辑和拖动；Windows 任务栏预览与 Alt-Tab 只做回归确认，不改变既有逻辑。
